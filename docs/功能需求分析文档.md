# 毕业论文管理系统功能需求分析

## 1. 系统概述

毕业论文管理系统是一个专为高校设计的信息化平台，用于管理学生毕业论文的整个生命周期，包括师生互选、选题、任务书下达、开题、中期检查、论文撰写、评审等环节。系统将实现论文管理的数字化、规范化和流程化，提高管理效率，确保论文质量。

## 2. 角色分析

### 2.1 管理员
- **核心职责**：系统配置与管理、用户管理、数据统计与分析
- **权限范围**：全系统权限，包括所有功能模块的访问与操作权限
- **系统入口**：管理员登录后进入管理员专属系统界面
- **注册方式**：无需注册，由系统直接分配账号和密码

### 2.2 教师
- **核心职责**：师生互选管理、课题审核、任务书下达、论文指导、文档审核、查看学生进度
- **权限范围**：管理自己的指导学生、审核论文相关材料、上传下载文件、查看学生论文进度
- **系统入口**：教师登录后进入教师专属系统界面
- **注册方式**：注册时输入工号、姓名和角色

### 2.3 学生
- **核心职责**：师生互选、确定选题、提交开题报告、中期检查、论文撰写与提交、查看个人进度
- **权限范围**：管理个人论文相关事务，上传下载文件，查看审核状态，查看个人论文进度
- **系统入口**：学生登录后进入学生专属系统界面
- **注册方式**：注册时输入学号、姓名和角色

## 3. 功能模块划分

### 3.1 系统管理模块
- 用户管理
- 角色与权限管理
- 系统配置（各阶段截止时间设置）
- 数据备份与恢复

### 3.2 注册与登录模块
- 教师注册（工号、姓名、角色）
- 学生注册（学号、姓名、角色）
- 管理员登录（直接分配账号密码）
- 个人信息完善 （补充院系、专业、联系方式等，支持后期修改）

### 3.3 师生互选模块
- 导师信息管理（教师可查看、添加、删除、修改指导名额）
- 学生选择导师 （学生按院系专业可查看导师与剩余指导名额、选择导师，每个学生最多选择一个导师）
- 导师审核学生申请（教师可查看、接受或拒绝学生申请）
- 互选结果管理 （系统展示互选结果，管理员查看整体互选情况，处理异常情况）
- 绑定关系解除与重新选择 （互选截止前可自由解除，截止后需要管理审核方可解除）

### 3.4 论文管理模块
- 选题管理 （学生提交选题、教师审核选题、选题修改与重新提交）
- 任务书管理 （教师可上传任务书文件给学生，学生可下载查看）
- 开题管理 （学生提交开题报告、教师审核开题报告、开题修改与重新提交）
- 中期检查管理 （学生提交中期检查报告、教师审核中期检查报告、中期检查修改与重新提交）
- 论文管理 （学生提交论文、教师审核论文、论文修改与重新提交）
- 进度跟踪管理 （教师可查看学生的论文进度，包括选题、开题、中期检查、论文提交等环节）

### 3.5 文件管理模块
- 文件上传 （支持PDF、Word等格式，设置文件大小限制，校验文件合法性）
- 文件同步（所有上传文件自动同步到绑定的师生，同步状态提示）
- 文件下载
- 文件版本管理
- 文件预览

### 3.6 统计分析模块
- 论文统计
- 评审统计
- 导出报表

## 4. 详细功能需求

### 4.1 管理员功能
1. **用户管理**
   - 新增、编辑、删除用户账号，支持批量导入/导出用户信息（对接教务系统数据）
   - 重置用户密码
   - 查看用户列表与详情，筛选不同角色、院系的用户

2. **角色与权限管理**
   - 角色创建与管理
   - 权限分配与调整
   - 角色关联用户

3. **系统配置**
   - 各阶段截止时间设置（选题、开题、中期检查、论文提交等）- 仅作为通知提醒作用，不限制实际操作
   - 通知公告管理（仅管理员可发布，用于系统公告和截止时间提醒）
   - 系统参数配置

4. **数据管理**
   - 数据备份
   - 数据恢复
   - 数据导入/导出

5. **统计分析**
   - 查看各模块统计数据
   - 生成统计报表
   - 导出分析结果

### 4.2 教师功能
1. **注册与登录**
   - 教师注册（输入工号、姓名、角色）
   - 登录系统
   - 完善个人信息

2. **师生互选管理**
   - 查看学生申请列表
   - 接受或拒绝学生申请
   - 查看已确认的指导学生，管理指导学生列表
   - 解除与学生的绑定关系（互选截止前可直接解除，截止后需提交申请，由管理员审核）
   - 设置指导学生人数上限

3. **论文管理**
   - 审核学生选题（通过/不通过），填写审核意见（同步给学生，方便学生修改）
   - 上传任务书文件给绑定的学生，自动同步到学生端
   - 审核开题报告（通过/不通过）
   - 审核中期检查报告（通过/不通过）
   - 审核学生论文（通过/不通过）
   - 查看自己所有学生的论文进度

4. **文件管理**
   - 上传相关文件给学生（自动同步到绑定学生）
   - 下载学生提交的文件
   - 查看文件版本历史
   - 接收学生上传后同步的文件

### 4.3 学生功能
1. **注册与登录**
   - 学生注册（输入学号、姓名、角色）
   - 登录系统
   - 完善个人信息，填写院系、专业、联系方式（可后期修改）
   - 注册后首次登录必须选择导师，未完成导师选择无法进入论文管理模块

2. **师生互选**
   - 按院系和专业筛选导师
   - 查看导师详情与指导名额
   - 提交导师申请
   - 查看申请状态（待审核/已接受/已拒绝）
   - 解除与导师的绑定关系 （互选截止前可直接解除，截止后需提交申请，由管理员审核）
   - 重新选择导师 （解除绑定后，在互选截止前学生可以重新选择导师；截止后重新选择需要管理员审核）

3. **论文管理**
   - 提交论文选题给绑定的老师
   - 查看选题审核结果
   - 查看教师上传的任务书文件，下载任务书
   - 提交开题报告，填写开题相关信息，上传开题文件
   - 查看开题审核结果，审核不通过时修改并重新提交
   - 提交中期检查报告，填写中期检查相关信息，上传中期检查文件
   - 查看中期检查审核结果，审核不通过时修改并重新提交
   - 提交论文，填写论文相关信息，上传论文文件
   - 查看论文审核结果，审核不通过时重新修改并提交
   - 查看个人论文进度，接收各阶段截止时间提醒和逾期预警

4. **文件管理**
   - 上传各类文档（选题、开题报告、中期检查、论文）（自动同步给绑定导师）
   - 下载教师上传的文件（任务书）
   - 查看文件版本历史
   - 修改后重新上传文件并同步给教师

## 5. 核心业务流程

### 5.1 注册与登录流程
1. **学生注册**：
   - 输入学号、姓名、角色
   - 登录系统，完善个人信息（院系、专业、班级等）
   - 首次登录必须进入师生互选模块选择导师，完成绑定后方可进入其他模块

2. **教师注册**：
   - 输入工号、姓名、角色
   - 登录系统
   - 完善个人信息（院系，专业）
   - 设置指导学生人数上限

3. **管理员登录**：
   - 使用系统分配的账号密码登录
   - 进入管理员系统界面，完成系统初始化配置（截止时间、参数设置等）

### 5.2 师生互选流程
1. 管理员设置互选时间和各阶段截止时间
2. 教师设置指导学生人数上限
3. 学生按院系和专业筛选导师
4. 学生提交导师申请
5. 导师查看申请
6. 导师接受或拒绝学生申请
7. 学生查看申请状态，若被拒绝，可重新选择导师
8. 系统确认互选结果，建立绑定关系，同步给双方
9. 绑定关系建立后，文件上传同步和审核关系仅在该师生之间进行
10. 互选截止时间前，师生可以自由解除绑定
11. 互选截止时间后，师生解除绑定需要管理员审核
12. 解除绑定后，学生可以重新选择导师
13. 互选截止时间后，学生重新选择导师也需要管理员审核
14. 新的绑定关系建立后，文件上传同步和审核关系仅在新的师生之间进行

### 5.3 论文管理流程
1. 学生提交论文选题给绑定的老师
2. 导师审核选题（通过/不通过），填写审核意见并同步给学生
3. 选题通过后，导师上传任务书文件给绑定的学生（自动同步给学生）
4. 学生下载查看任务书文件并根据任务书撰写开题报告
5. 学生上传开题报告文件（自动同步给绑定导师）
6. 导师下载查看并审核开题报告（通过/不通过）
7. 若不通过，学生修改并重新上传，文件自动同步给导师
8. 开题通过后，学生可以同时进行：
   - 撰写并上传中期检查报告（自动同步给绑定导师）
   - 撰写并上传论文（自动同步给绑定导师）
9. 导师下载查看并审核中期检查报告和论文（通过/不通过）
10. 若不通过，学生修改并重新上传，文件自动同步给导师
11. 导师最终审核通过

### 5.4 文件同步流程
1. 绑定关系建立
2. 学生或教师上传文件
3. 系统自动识别绑定关系
4. 文件自动同步到绑定的教师或学生账户
5. 接收方收到文件同步通知
6. 接收方可以下载查看同步的文件

## 6. 数据库设计建议

### 6.1 核心数据表
1. **用户表** (users)
   - 用户ID、用户名（学号/工号）、密码、姓名、角色、院系、专业、联系方式等

2. **角色权限表** (roles_permissions)
   - 角色ID、角色名称、权限列表等

3. **系统设置表** (system_settings)
   - 设置ID、阶段名称、开始时间、截止时间等

4. **师生互选表** (teacher_student_selection)
   - 互选ID、学生ID、教师ID、申请时间、状态（待审核/已接受/已拒绝/已解除）、审核时间、绑定时间、解除时间等

5. **选题表** (topic_selections)
   - 选题ID、学生ID、教师ID、选题名称、选题描述、提交时间、审核状态（通过/不通过）、审核意见、审核时间等

6. **任务书表** (assignment_tasks)
   - 任务书ID、选题ID、学生ID、教师ID、文件路径、下达时间、截止时间等

7. **开题报告表** (proposal_reports)
   - 报告ID、选题ID、学生ID、教师ID、文件路径、提交时间、审核状态（通过/不通过）、审核意见、审核时间等

8. **中期检查表** (midterm_reports)
   - 报告ID、选题ID、学生ID、教师ID、文件路径、提交时间、审核状态（通过/不通过）、审核意见、审核时间等

9. **论文表** (papers)
   - 论文ID、选题ID、学生ID、教师ID、文件路径、提交时间、审核状态（通过/不通过）、审核意见、审核时间等

10. **文件表** (files)
    - 文件ID、关联ID、文件名称、文件路径、上传人ID、接收人ID、上传时间、文件类型、版本号、同步状态等

11. **进度表** (progress)
    - 进度ID、学生ID、教师ID、选题状态、任务书状态、开题状态、中期检查状态、论文状态、更新时间等

12. **通知表** (notifications)
    - 通知ID、标题、内容、发布人ID、发布时间、接收角色等

### 6.2 关系设计
- 用户与角色权限：多对一
- 用户与师生互选：一对多
- 用户与选题：一对多
- 选题与任务书：一对一
- 选题与开题报告：一对一
- 选题与中期检查表：一对一
- 选题与论文：一对一
- 选题/开题报告/中期检查表/论文与文件：一对多
- 学生与进度：一对一

## 7. 技术选型建议

### 7.1 前端技术
- **框架**：Vue.js / React
- **UI组件库**：Element Plus / Ant Design
- **开发工具**：VS Code
- **构建工具**：Webpack / Vite

### 7.2 后端技术
- **语言**：Java / Python / Node.js
- **框架**：Spring Boot / Django / Express
- **数据库**：MySQL / PostgreSQL
- **缓存**：Redis（可选）
- **认证**：JWT / Session

### 7.3 部署方案
- **服务器**：Linux
- **容器化**：Docker（可选）
- **持续集成**：Jenkins（可选）

### 7.4 其他考虑
- **安全性**：实现HTTPS、数据加密、权限控制
- **可扩展性**：模块化设计，支持后续功能扩展
- **兼容性**：支持主流浏览器
- **响应式设计**：适配PC端与移动端

## 8. 系统特色功能建议

1. **智能导师匹配**：基于学生专业、兴趣与导师研究方向，推荐合适的导师
2. **进度跟踪**：可视化展示论文各阶段进度，及时提醒逾期任务
3. **文件版本管理**：自动保存文件历史版本，方便查看修改记录
4. **自动文件同步**：所有上传文件自动同步到绑定的师生账户，确保信息实时共享
5. **师生绑定管理**：支持绑定关系的建立、解除与重新选择
6. **数据统计**：多维度分析论文质量、导师工作量等数据
7. **移动应用**：开发配套移动应用，方便随时随地查看与处理任务
8. **批量操作**：教师可以对多个学生进行批量管理操作

## 9. 实施建议

1. **需求确认**：与高校相关部门深入沟通，确认具体需求细节
2. **原型设计**：开发系统原型，进行用户体验测试
3. **开发与测试**：分阶段开发，进行充分的功能测试与性能测试
4. **培训与上线**：对管理员、教师与学生进行系统培训，然后正式上线
5. **运维与优化**：建立运维机制，持续收集用户反馈并进行系统优化

# 哈尔滨金融学院毕业论文管理系统研究描述

## 1. 系统研究背景与目标

本次研究旨在设计并实现一套哈尔滨金融学院毕业论文管理系统，核心目的是解决学院传统毕业论文人工管理的痛点，规范毕业论文全流程管理，满足管理员、指导教师、学生三类角色的差异化需求，实现毕业论文管理的数字化、规范化、高效化，贴合哈尔滨金融学院金融类专业毕业论文的培养要求与管理实际。本系统明确划分三类核心角色，各角色权责清晰、协同联动：管理员负责系统整体管控与用户管理，指导教师负责论文指导与审核，学生负责完成论文相关材料提交与进度跟进。

## 2. 系统架构设计

本研究聚焦哈尔滨金融学院毕业论文管理系统的整体架构，明确系统的核心组成部分，包括前端交互层、后端逻辑层、数据存储层三大模块，确保各层级协同高效、数据互通。

### 2.1 前端交互层

前端采用Vue 3作为核心框架，结合Element Plus组件库构建响应式用户界面。前端架构采用单页应用（SPA）模式，通过Vue Router实现不同角色界面的路由切换，使用Pinia进行状态管理，通过Axios与后端RESTful API进行数据交互。前端主要包含三类角色的专属界面：

- **管理员界面**：包含仪表盘、用户管理、师生互选管理、系统设置、通知公告等模块
- **教师界面**：包含仪表盘、学生管理、选题管理、任务书管理、报告管理、论文管理等模块
- **学生界面**：包含仪表盘、选择导师、申请状态、论文选题、任务书、开题报告、中期检查、最终论文等模块

### 2.2 后端逻辑层

后端采用Spring Boot 3.2.0框架，基于Java 17开发，采用分层架构设计，包括Controller层、Service层、Mapper层。后端主要功能模块包括：

- **认证授权模块**：基于Spring Security和JWT实现用户认证与权限控制
- **用户管理模块**：处理用户注册、登录、信息完善等功能
- **师生互选模块**：实现导师选择、申请审核、绑定关系管理等功能
- **论文流程模块**：管理选题、任务书、开题报告、中期检查、论文等全流程
- **文件管理模块**：实现文件上传、下载、版本管理等功能
- **进度统计模块**：提供学生进度跟踪、教师统计、数据可视化等功能
- **系统管理模块**：包括系统设置、通知公告等功能

后端使用MyBatis-Plus作为ORM框架，简化数据库操作，提高开发效率。

### 2.3 数据存储层

数据存储层采用MySQL 8.0关系型数据库，设计了完整的数据表结构，主要包括：

- **用户表（users）**：存储管理员、教师、学生的基本信息
- **师生互选表（teacher_student_selection）**：记录师生申请与绑定关系
- **选题表（topic_selections）**：存储论文选题信息与审核状态
- **任务书表（assignment_tasks）**：管理教师下达的任务书
- **开题报告表（proposal_reports）**：存储开题报告信息与审核状态
- **中期检查表（midterm_reports）**：存储中期检查信息与审核状态
- **论文表（papers）**：存储最终论文信息与审核状态
- **文件表（files）**：管理所有上传的文件信息
- **进度表（progress）**：记录学生论文各阶段进度
- **系统设置表（system_settings）**：存储系统配置参数
- **通知表（notifications）**：管理系统通知公告

研究过程中，结合国内外成熟架构经验，优化架构设计，兼顾系统的稳定性、可扩展性与安全性，适配学院毕业论文管理的高峰期需求，为后续系统开发奠定坚实的架构基础，确保系统能够长期稳定运行并支持后续功能拓展。

## 3. 核心功能组件

研究针对哈尔滨金融学院毕业论文管理系统的功能组件，围绕管理员、指导教师、学生三类核心角色展开。

### 3.1 管理员功能组件

管理员负责系统整体管控与用户管理，主要功能包括：

- **用户信息管理**：新增、编辑、删除用户账号，查看用户列表与详情，重置用户密码
- **师生互选管理**：查看整体互选情况，处理异常情况
- **系统设置**：配置系统参数，管理各阶段截止时间
- **通知公告**：发布系统公告和截止时间提醒
- **数据统计**：查看各模块统计数据，生成统计报表

### 3.2 指导教师功能组件

指导教师负责论文指导与审核，主要功能包括：

- **师生互选管理**：查看学生申请列表，接受或拒绝学生申请，查看已确认的指导学生
- **选题管理**：审核学生选题（通过/不通过），填写审核意见
- **任务书管理**：上传任务书文件给绑定的学生
- **报告管理**：审核开题报告、中期检查报告（通过/不通过）
- **论文管理**：审核学生论文（通过/不通过）
- **进度跟踪**：查看自己所有学生的论文进度，包括选题、开题、中期检查、论文提交等环节的实时可视化进度
- **仪表盘**：可视化展示学生信息和工作进度，包括已完成任务、任务进度等实时数据

### 3.3 学生功能组件

学生负责完成论文相关材料提交与进度跟进，主要功能包括：

- **注册登录**：学生注册（输入学号、姓名、角色），登录系统，完善个人信息
- **师生互选**：按院系和专业筛选导师，查看导师详情与指导名额，提交导师申请，查看申请状态
- **论文选题**：提交论文选题给绑定的老师，查看选题审核结果
- **任务书**：查看教师上传的任务书文件，下载任务书
- **开题报告**：提交开题报告，填写开题相关信息，上传开题文件，查看开题审核结果
- **中期检查**：提交中期检查报告，填写中期检查相关信息，上传中期检查文件，查看中期检查审核结果
- **最终论文**：提交论文，填写论文相关信息，上传论文文件，查看论文审核结果
- **进度跟踪**：查看个人论文进度，实时可视化展示各阶段状态，包括已完成任务、任务进度等实时数据

系统实现了严格的流程控制：学生必须先与老师绑定后才能进行后续功能；学生上传开题报告等功能需要在收到老师任务书之后才能进行；未绑定导师的学生提交开题报告、中期检查、论文时会弹出警告提示，只有绑定之后提交才会弹窗提交成功。

为评估这些功能组件的完整性与实用性，契合毕业论文管理的实际需求，系统实施了全面的功能测试，验证功能组件的适配性与便捷性。

## 4. 技术实施

技术实施层面，通过代码审计、效能检测等手段，评估系统技术选型的合理性、领先性与稳定性，排查功能漏洞与运行隐患。

### 4.1 前端技术栈

- **框架**：Vue 3.x，采用Composition API，轻量级框架，响应式设计，组件化开发
- **UI组件库**：Element Plus 2.x，基于Vue 3的组件库，提供丰富的UI组件
- **状态管理**：Pinia 2.x，Vue 3官方推荐的状态管理库
- **路由**：Vue Router 4.x，支持嵌套路由和路由守卫
- **构建工具**：Vite 4.x，快速的前端构建工具，热更新速度快
- **HTTP客户端**：Axios 1.x，支持Promise API，拦截器功能强大

### 4.2 后端技术栈

- **语言**：Java 17，编译型语言，性能稳定，生态成熟
- **框架**：Spring Boot 3.2.0，快速开发框架，内置Tomcat，自动配置
- **安全框架**：Spring Security 6.x，强大的安全框架，支持认证和授权
- **ORM框架**：MyBatis-Plus 3.x，基于MyBatis的增强工具，简化CRUD操作
- **认证方式**：JWT（JSON Web Token），无状态认证，便于水平扩展
- **任务调度**：Spring Task，轻量级任务调度框架
- **构建工具**：Maven，项目管理和构建工具

### 4.3 数据库技术

- **主数据库**：MySQL 8.0，关系型数据库，稳定可靠，适合存储结构化数据
- **密码加密**：BCrypt，强哈希算法，适合密码加密存储

### 4.4 系统特色功能实现

- **进度实时可视化**：系统实现了进度权重计算（选题10%、任务书15%、开题30%、中期25%、论文20%），所有显示进度的地方都实时可视化，及时更新同步，工作台的已完成任务、任务进度采用实时数据，而非假数据
- **文件版本管理**：实现了文件版本管理功能，支持文件的上传、下载和版本追溯
- **自动文件同步**：所有上传文件自动同步到绑定的师生账户
- **师生绑定管理**：支持绑定关系的建立、解除与重新选择
- **定时自动刷新**：前端实现了30秒自动刷新机制，确保数据实时更新
- **进度检查服务**：创建了完整的ProgressCheckService，包含getStudentStats、getTeacherStats、getStudentProgressDetail等统计方法

## 5. 研究总结

考虑到论文结构、技术水平等局限，本文将重点放在毕业论文管理系统在设计、开发、实施、测试等方面的详细论述，确保研究重点突出、贴合实际，为系统的落地应用提供切实的技术支撑。本系统通过前后端分离架构，实现了哈尔滨金融学院毕业论文管理的数字化、规范化、高效化，满足了管理员、指导教师、学生三类角色的差异化需求，为学院金融类专业毕业论文的培养与管理提供了有力的技术保障。

# 毕业设计管理系统详细设计与实现文档

## 1. 系统概述

本系统是一个毕业设计管理系统，用于管理学生的毕业设计全过程，包括选题、师生匹配、进度跟踪、论文提交与评审等功能。系统采用分层架构，包括控制层、服务层、数据访问层和实体层，实现了完整的业务流程管理。

系统主要功能模块包括：
- 用户认证功能：用户注册、登录、权限管理
- 师生互选功能：学生申请导师、教师审核申请、师生绑定
- 选题管理功能：学生提交选题、教师审核选题、任务书上传
- 论文管理功能：开题报告、中期检查、最终论文提交与评审
- 文件管理功能：文件上传、下载、版本管理
- 系统管理功能：系统设置、通知管理、用户管理

## 2. 系统架构

### 2.1 技术栈

- **前端**：Vue 3、Vue Router、Pinia、Axios
- **后端**：Spring Boot 3、MyBatis-Plus、Spring Security、JWT
- **数据库**：MySQL
- **认证**：JWT 令牌认证
- **文件存储**：本地文件系统

### 2.2 分层架构

```mermaid
classDiagram
    direction LR
    
    %% 前端层
    Frontend --> Controller
    
    %% 后端层
    Controller --> Service

    Controller --> JwtTokenProvider

    %% 层间关系

    }
```

## 3. 核心实体类设计

### 3.1 User（用户）

**职责**：存储系统用户信息，包括学生、教师和管理员。

**属性**：
- id: Long - 用户ID
- username: String - 用户名
- password: String - 密码（加密存储）
- name: String - 真实姓名
- role: String - 角色（student/teacher/admin）
- department: String - 部门
- major: String - 专业
- contact: String - 联系方式
- maxStudents: Integer - 教师最大指导学生数
- createdAt: LocalDateTime - 创建时间
- updatedAt: LocalDateTime - 更新时间

**关系**：
- 与TopicSelection：一对多（一个学生可以提交多个选题）
- 与TeacherStudentSelection：一对多（一个学生可以申请多个教师，一个教师可以被多个学生申请）
- 与Progress：一对多（一个学生有多个进度记录）
- 与Paper：一对多（一个学生提交一篇论文，一个教师评审多篇论文）

### 3.2 TopicSelection（选题）

**职责**：管理学生的选题信息，包括选题名称、描述、评审状态等。

**属性**：
- topicId: Long - 选题ID
- studentId: Long - 学生ID
- teacherId: Long - 教师ID
- topicName: String - 选题名称
- topicDescription: String - 选题描述
- reviewStatus: String - 评审状态（PENDING/APPROVED/REJECTED）
- reviewOpinion: String - 评审意见
- submitTime: Date - 提交时间
- reviewTime: Date - 评审时间
- createdAt: Date - 创建时间
- updatedAt: Date - 更新时间

**关系**：
- 与User：多对一（多个选题属于一个学生和一个教师）
- 与Paper：一对一（一个选题对应一篇论文）

### 3.3 TeacherStudentSelection（师生选择）

**职责**：管理学生与教师之间的双向选择过程，包括申请、审批、绑定等状态。

**属性**：
- id: Long - 选择ID
- studentId: Long - 学生ID
- teacherId: Long - 教师ID
- status: String - 状态（申请中/已批准/已绑定/已解绑）
- applyTime: LocalDateTime - 申请时间
- reviewTime: LocalDateTime - 评审时间
- bindTime: LocalDateTime - 绑定时间
- unbindTime: LocalDateTime - 解绑时间
- createdAt: LocalDateTime - 创建时间
- updatedAt: LocalDateTime - 更新时间

**关系**：
- 与User：多对一（多个选择记录属于一个学生和一个教师）

### 3.4 Progress（进度）

**职责**：跟踪学生毕业设计的各个阶段进度。

**属性**：
- id: Long - 进度ID
- studentId: Long - 学生ID

- status: String - 状态（未提交/已提交/已通过/未通过）

- updatedAt: LocalDateTime - 更新时间

**关系**：
- 与User：多对一（多个进度记录属于一个学生）

## 4. 服务类设计

### 4.1 UserService

**职责**：处理用户相关的业务逻辑，如用户注册、登录验证、用户信息管理等。

**方法**：
- generatePasswordHash(String password): String - 生成密码哈希
- findByUsername(String username): User - 根据用户名查找用户
- findById(Long id): User - 根据ID查找用户
- register(User user): User - 注册新用户
- update(User user): void - 更新用户信息
- getAllUsers(): List<User> - 获取所有用户
- getTeachers(): List<User> - 获取所有教师
- getTeachersByDepartmentAndMajor(String department, String major): List<User> - 根据部门和专业获取教师
- deleteUser(Long id): void - 删除用户
- addUser(User user): User - 添加用户

### 4.2 TeacherStudentSelectionService

**职责**：处理师生匹配相关的业务逻辑，如学生申请教师、教师审批等。

**方法**：
- studentSelectTeacher(Long studentId, Long teacherId): void - 学生选择教师
- teacherAcceptSelection(Long selectionId): void - 教师接受申请
- teacherRejectSelection(Long selectionId): void - 教师拒绝申请
- adminApproveSelection(Long selectionId): void - 管理员审批申请
- getSelectionsWithStudentInfoByTeacherId(Long teacherId): List<SelectionWithStudentInfo> - 获取教师收到的申请（包含学生信息）
- getSelectionsWithTeacherInfoByStudentId(Long studentId): List<SelectionWithTeacherInfo> - 获取学生的申请记录（包含教师信息）
- getPendingSelections(): List<TeacherStudentSelection> - 获取待审批的申请
- isTeacherFull(Long teacherId): boolean - 检查教师是否已满额
- hasPendingApplication(Long studentId): boolean - 检查学生是否有待审批的申请

### 4.3 TopicSelectionService

**职责**：处理选题相关的业务逻辑，如选题提交、评审等。

**方法**：
- submitTopic(TopicSelection topicSelection): void - 学生提交选题
- approveTopic(Long topicId): void - 教师审核通过选题
- rejectTopic(Long topicId, String comment): void - 教师审核拒绝选题
- uploadTask(Long topicId, Long taskFileId): void - 教师上传任务书
- getPendingTopicsByTeacherId(Long teacherId): List<TopicSelection> - 获取教师的待审核选题
- getTopicStatus(Long studentId): Map<String, Object> - 获取学生的选题状态
- getTaskFileId(Long topicId): Long - 获取任务书文件ID

### 4.4 ProgressService

**职责**：处理进度相关的业务逻辑，如进度更新、查询等。

**方法**：
- updateProgress(Long studentId, String stage, String status): void - 更新进度
- getProgressByStudentId(Long studentId): List<Progress> - 获取学生的进度记录
- checkProgressStatus(Long studentId, String stage): String - 检查进度状态

### 4.5 PaperService

**职责**：处理论文相关的业务逻辑，如论文提交、评审、评分等。

**方法**：
- submitPaper(Paper paper): void - 学生提交论文
- reviewPaper(Long paperId, String reviewStatus, String reviewOpinion, Integer score): void - 教师评审论文
- getPaperByStudentId(Long studentId): Paper - 获取学生的论文
- getPapersByTeacherId(Long teacherId): List<Paper> - 获取教师需要评审的论文
- updatePaperStatus(Long paperId, String status): void - 更新论文状态

### 4.6 FileService

**职责**：处理文件相关的业务逻辑，如文件上传、下载、管理等。

**方法**：
- uploadFile(MultipartFile file, Long uploaderId): File - 上传文件
- downloadFile(Long fileId): File - 下载文件
- deleteFile(Long fileId): void - 删除文件
- getFileById(Long fileId): File - 根据ID获取文件
- getFilesByUploaderId(Long uploaderId): List<File> - 获取上传者的文件

### 4.7 NotificationService

**职责**：处理通知相关的业务逻辑，如通知发送、查询等。

**方法**：
- sendNotification(String title, String content, Long recipientId, Long senderId): void - 发送通知
- getNotificationsByRecipientId(Long recipientId): List<Notification> - 获取接收者的通知
- markAsRead(Long notificationId): void - 标记通知为已读
- getUnreadCount(Long recipientId): Integer - 获取未读通知数量

### 4.8 SystemSettingService

**职责**：处理系统设置相关的业务逻辑，如设置修改、查询等。

**方法**：
- getSetting(String key): String - 获取系统设置
- updateSetting(String key, String value): void - 更新系统设置
- getAllSettings(): List<SystemSetting> - 获取所有系统设置

- 注册页面：用户填写注册信息

5. 返回用户信息和令牌

**注册流程**：
1. 用户填写注册信息

3. 后端验证注册信息

5. 保存用户信息

5. 提交申请：选择心仪的教师，提交申请
6. 系统验证：
   - 检查学生是否已经绑定导师
   - 检查学生是否有待审批的申请
   - 检查教师是否已满额
7. 生成申请记录：系统生成师生互选申请记录，状态为"PENDING"

5. 系统处理：
   - 若教师接受申请，建立师生指导关系
   - 若教师拒绝申请，学生可以重新选择其他教师

#### 7.3.4 技术实现

**前端实现**：
- 选题提交页面：学生填写选题名称和描述
- 选题审核页面：教师查看和审核学生选题
- 任务书上传页面：教师上传任务书
- 选题状态页面：学生查看选题状态和任务书

**后端实现**：
- TopicSelectionController：处理选题相关的请求
- TopicSelectionService：处理选题的业务逻辑
- TopicSelectionMapper：数据访问
- AssignmentTaskMapper：任务书数据访问

**学生选题提交流程**：
1. 学生登录系统
2. 进入选题提交页面
3. 填写选题名称和描述
4. 提交选题：前端调用 `/topic/student/submit` 接口
5. 系统验证：检查学生是否已绑定导师
6. 保存选题：系统保存选题信息，状态为"PENDING"
7. 提示提交成功：显示提交成功提示，等待教师审核

**教师选题审核流程**：
1. 教师登录系统
2. 进入选题审核页面
3. 查看待审核选题：前端调用 `/topic/teacher/pending` 接口
4. 审核选题：
   - 审核通过：点击通过按钮，前端调用 `/topic/teacher/approve` 接口，系统更新选题状态为"APPROVED"，创建任务记录
   - 审核拒绝：填写审核意见，点击拒绝按钮，前端调用 `/topic/teacher/reject` 接口，系统更新选题状态为"REJECTED"，保存审核意见
5. 提示审核结果：显示审核成功提示

**任务书上传流程**：
1. 教师登录系统
2. 进入任务书上传页面
3. 选择已通过审核的选题
4. 上传任务书：前端先调用文件上传接口，获取文件ID，然后调用 `/topic/teacher/upload-task` 接口
5. 系统处理：查找与选题关联的任务记录，更新任务记录，设置任务书文件ID
6. 提示上传成功：显示上传成功提示

## 9. 技术实现细节

### 9.1 JWT认证实现

系统使用JWT（JSON Web Token）进行身份认证，具体实现如下：

1. **认证流程**：
   - 用户登录时，系统验证用户名和密码
   - 验证通过后，生成JWT令牌，包含用户ID、用户名和角色
   - 将令牌返回给前端
   - 前端存储令牌，用于后续请求
   - 后续请求时，前端在请求头中携带令牌
   - 后端验证令牌的有效性
   - 验证通过后，处理请求

2. **JWT配置**：
   - 令牌有效期：24小时
   - 签名算法：HS256
   - 密钥：系统配置的密钥

3. **安全措施**：
   - 密码加密存储：使用BCrypt加密
   - 令牌验证：验证令牌的签名和有效期
   - 权限控制：根据用户角色控制访问权限

### 9.2 文件存储实现

系统使用本地文件系统存储文件，具体实现如下：

1. **文件存储路径**：
   - 文件存储在服务器的指定目录下
   - 目录结构：`{basePath}/{year}/{month}/{day}/{fileId}`
   - basePath：系统配置的基础路径

2. **文件上传流程**：
   - 前端选择文件并上传
   - 后端接收文件，生成唯一文件ID
   - 创建存储目录
   - 保存文件到存储目录
   - 记录文件信息到数据库
   - 返回文件ID给前端

3. **文件下载流程**：
   - 前端请求下载文件，携带文件ID
   - 后端根据文件ID查找文件信息
   - 读取文件内容
   - 设置响应头，返回文件内容
   - 浏览器下载文件

### 9.3 事务管理

系统使用Spring的事务管理机制，确保业务操作的原子性，具体实现如下：

1. **事务配置**：
   - 使用`@Transactional`注解标记需要事务管理的方法
   - 事务隔离级别：默认隔离级别
   - 事务传播行为：默认传播行为

2. **事务管理场景**：
   - 选题提交和审核：确保选题状态和任务记录的一致性
   - 论文提交和评审：确保论文状态和评审信息的一致性
   - 师生互选：确保申请状态和师生关系的一致性

### 9.4 异常处理

系统实现了统一的异常处理机制，确保系统的稳定性和可靠性，具体实现如下：

1. **异常处理策略**：
   - 使用`@RestControllerAdvice`和`@ExceptionHandler`注解处理异常
   - 分类处理不同类型的异常
   - 返回统一的错误响应格式

2. **异常类型**：
   - 业务异常：如教师已满额、学生已绑定导师等
   - 系统异常：如数据库错误、文件存储错误等
   - 认证异常：如令牌无效、权限不足等

3. **错误响应格式**：
   ```json
   {
     "code": 错误码,
   ```

### 10.3 未来展望

随着教育信息化的不断发展，毕业设计管理系统将在以下方面得到进一步的发展：

1. **智能化**：利用人工智能技术，实现智能选题推荐、智能论文评审等功能
2. **移动化**：开发移动端应用，支持学生和教师随时随地访问系统
3. **集成化**：与其他教育管理系统集成，实现数据共享和业务协同
4. **国际化**：支持多语言和国际化教育模式，适应全球化教育的需求

通过不断的优化和改进，毕业设计管理系统将成为教育信息化的重要组成部分，为高校的毕业设计管理提供更加便捷、高效、智能的解决方案。
